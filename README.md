## 🏗 Key Project Architecture

My projects typically follow this **ELT** (Extract, Load, Transform) pattern:

1.  **Ingestion:** Raw data is pulled from **SQL Server** or **APIs**.
2.  **Orchestration:** **Astro/Airflow** manages the flow and triggers tasks in **Docker** containers.
3.  **Loading:** Data is landed into **PostgreSQL** (Operational) or **Snowflake** (Analytical).
4.  **Transformation:** **dbt** runs models to turn raw data into clean **Star Schemas** (Facts & Dimensions).
5.  **Quality Control:** Automated data testing via dbt and Airflow sensors.
